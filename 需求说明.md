# 口算练习界面 - 需求说明

## 一、你已经提到的需求（整理版）

### 1. 题型选项（入口选择）
- 两位数 + 两位数
- 两位数 + 三位数
- 三位数 + 三位数
- 一位数 × 两位数
- 两位数 × 两位数

### 2. 做题流程
- 选完题型后：**随机出题**，一题一题做，**不设轮次**，做到不想做为止
- **答对**：**即时**随机奖励贴纸（每答对一题就给一张）
- **答错**：给机会重试，累计 **5 次错误** 后显示正确答案，然后进入下一题

### 3. 贴纸系统
- 贴纸存入 **贴纸库**
- 可以 **点击查看** 贴纸库里的贴纸

### 4. 奖励规则
- **每答对一题即给一张贴纸**（即时奖励）；是否再按答对题数设额外档位奖励见下文“需要你定的”

## 三、技术栈建议（需保存数据、安卓浏览器）

在「需要保存数据」且主要在安卓手机浏览器打开的前提下，推荐如下方案：

| 方面 | 推荐方案 | 说明 |
|------|----------|------|
| **前端** | **纯 HTML + CSS + JavaScript**（无构建） | 单页即可，安卓 Chrome 兼容好；无需 Node/npm，部署就是传静态文件，维护简单。 |
| **数据保存** | **localStorage** | 贴纸库、进度、历史等存浏览器本地，下次打开仍在；无需后端和数据库。 |
| **运行方式** | **浏览器打开网页**，可选 **PWA** | 用 Chrome 打开 URL 即可；加主屏幕/离线可做 PWA。 |
| **贴纸** | 图片 URL 或 emoji + **存 ID 列表到 localStorage** | 贴纸库用数组存已获得的贴纸 ID，页面根据 ID 渲染；“保存到相册”用 `<a download>` 或 canvas 转图片触发下载。 |
| **出题** | 前端 JS 按题型随机生成数字 | 加法/乘法按位数范围随机，结果现算即可，无需接口。 |

**不推荐**：React/Vue + 构建、Node 后端、数据库 —— 当前需求用纯静态页 + localStorage 即可满足，技术栈越简单越适合部署和后续修改。

### localStorage 说明：在哪里保存、在哪里找到、贴纸从哪里来

| 问题 | 说明 |
|------|------|
| **在哪里保存** | 数据保存在**用户手机/电脑的浏览器里**，和你打开的网站地址（域名）绑定。不是存在某个你能看到的文件夹，而是浏览器为这个网站分配的一块本地空间。代码里用 `localStorage.setItem('key', value)` 就会写入。 |
| **在哪里找到** | **用户**：不用找，只要再次用同一浏览器打开**同一个网址**，口算页面会自动读到之前的贴纸库、进度等。<br>**开发者**：在电脑 Chrome 里打开该页面 → 按 F12 → 选「Application / 应用」→ 左侧「Local Storage」→ 点你的网站地址，就能看到存的 key 和 value。安卓 Chrome 可用远程调试或 `chrome://inspect` 查看。 |
| **贴纸从哪里来 / 怎么“上传”** | 两种做法：<br>**做法 A（推荐）**：贴纸图片**提前放在项目里**（例如 `img/stickers/` 或使用 emoji），在代码里写好「贴纸列表」。用户答对题后，程序随机选一张，只把**这张贴纸的 ID** 存进 localStorage（如 `["s1","s3","s2"]`）。贴纸图不存进 localStorage，只是根据 ID 从项目资源里显示。<br>**做法 B**：若要做「用户自己上传图片当贴纸」，在口算应用里做一个**上传入口**（如贴纸库页的「添加我的贴纸」），用 `<input type="file">` 选图，转成 base64 或链接再存到 localStorage。注意 localStorage 约 5MB 限制，图片多容易满。 |

### 贴纸图片放在项目哪里、手机端怎么打开

| 问题 | 说明 |
|------|------|
| **贴纸文件保存在哪里** | 就保存在**你这个口算项目所在的文件夹**里。例如项目根目录是 `kousuan`，就建一个子文件夹 `img/stickers/`，把贴纸图片放进去，例如：<br>`kousuan/index.html`<br>`kousuan/img/stickers/star.png`、`heart.png`、`cup.png` …<br>代码里用相对路径引用，如 `img/stickers/star.png`。若用 **emoji**（如 ⭐🎉❤️），则不需要任何图片文件，直接写在 JS 里即可。 |
| **手机端网页怎么打开、能加载到贴纸吗** | 手机要能打开页面并看到贴纸，需要让手机访问到**同一套项目文件**（含 HTML 和 `img/stickers/`）。两种常见方式：<br>**方式 1：部署到网上** — 把整个 `kousuan` 项目（含 `index.html` 和 `img/stickers/`）上传到任意能访问的网站，例如 GitHub Pages、Netlify、或你自己的服务器。然后手机浏览器打开这个网址（如 `https://xxx.github.io/kousuan/`），页面和贴纸图片都会从该地址加载，手机端正常显示。<br>**方式 2：同 WiFi 下本地调试** — 在电脑上在 `kousuan` 目录用本地服务器跑（如 `npx serve` 或 VS Code Live Server），记下电脑的局域网 IP（如 `192.168.1.100:3000`），手机连同一 WiFi 后浏览器打开 `http://192.168.1.100:3000`，即可访问，贴纸也会从你这台电脑的 `img/stickers/` 被请求并显示在手机上。<br>总结：贴纸**保存在项目目录**；手机能打开的前提是**通过一个“地址”访问到这份项目**（要么已部署到公网/内网，要么同 WiFi 本地服务器）。 |

## 四、其他实现要点（供参考）

- **形式**：单页 HTML + CSS + JavaScript，部署到服务器或本地用浏览器打开。
- **安卓**：Chrome 打开该网页即可；若希望“像 App”可做成 **PWA**（可加主屏幕、离线缓存）。
- **贴纸**：可用 emoji、小图标或图片 URL；贴纸库用数组存 ID/配置，再同步到 localStorage。
- **出题**：根据题型随机生成符合范围的数，加法保证非负、乘法可限定在学过的范围内。
